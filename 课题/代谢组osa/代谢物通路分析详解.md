## 正文

### 1. 代谢物通路分析的定义与三类核心方法

本节概要：本节定义代谢物通路分析并说明其生物学意义，重点区分 ORA、QEA 和通路拓扑分析三类方法的输入、原理和适用场景。

#### 1.1 定义与假设

代谢物通路分析（Pathway Analysis）是将差异代谢物映射到已知生物学通路（biological pathways），通过统计方法评估哪些通路在实验条件下被显著扰动。

核心假设：生物体的表型变化不是由单个代谢物驱动，而是由一组功能相关的代谢物（即通路）协同变化所致。

通路分析解决三个实际问题：

- 差异代谢物数量多（数十至数百），需要按通路归类以提炼生物学叙事
- 通路级别的扰动比单个代谢物变化更稳定、可重复，且与疾病机制直接关联
- 通路作为桥梁可整合转录组、蛋白组数据进行多组学分析

#### 1.2 过表达分析（Over-Representation Analysis, ORA）

- 输入：差异代谢物列表（按 VIP > 1 或 p < 0.05 筛选）
- 方法：对每条通路执行 Fisher 精确检验，判断差异代谢物在该通路中的占比是否高于随机期望
- 统计量：超几何分布，P(X ≥ a) = Σ [C(P,i) × C(N-P, D-i)] / C(N, D)，其中 a 为通路中差异代谢物数，P 为通路代谢物总数，D 为差异代谢物总数，N 为背景集代谢物总数
- 校正：多重检验校正（Holm-Bonferroni / FDR / Bonferroni）
- 局限：仅使用"是否差异"的二元信息，忽略变化幅度和方向
- 代表工具：MetaboAnalyst、MBRole

#### 1.3 定量富集分析（Quantitative Enrichment Analysis, QEA）

- 输入：所有代谢物的定量信息（log2 fold change + p-value 或 t-score）
- 方法：使用 Globaltest 或 GlobalAncova，将代谢物按变化幅度排序后检验通路整体变化
- 优势：利用连续的变化幅度和方向信息，比 ORA 更灵敏，能发现效应较小但协同变化的通路
- 适用场景：深入分析阶段，作为 ORA 初筛后的定量验证

ORA 与 QEA 的关键区别：

| 维度 | ORA | QEA |
|------|-----|-----|
| 输入数据 | 差异代谢物列表 | 全代谢物定量矩阵 |
| 信息类型 | 二元（是否差异） | 连续（变化幅度 + 方向） |
| 灵敏性 | 较低，依赖 cutoff 选择 | 较高 |
| 适用场景 | 粗略筛选 | 深入分析、小效应通路发现 |

#### 1.4 通路拓扑分析（Pathway Topology Analysis）

- 核心思想：通路中不同位置的代谢物重要性不同，枢纽节点（hub node）的代谢物比末端节点的权重更大
- 中心性指标：
  - 度中心性（Degree Centrality）：与该节点直接相连的边数
  - 中介中心性（Betweenness Centrality）：经过该节点的最短路径数
  - 紧密度中心性（Closeness Centrality）：节点到所有其他节点平均最短距离的倒数
- 通路影响值（Pathway Impact）= Σ(命中代谢物中心性得分) / Σ(通路全部代谢物中心性得分)，取值 0-1
- 最终评分 = -log(p-value) × Pathway Impact，同时考虑统计显著性与生物网络位置重要性
- 可视化：气泡图，X 轴为 Pathway Impact，Y 轴为 -log10(p-value)，气泡大小表示命中代谢物数量；右上角的通路为显著且位于关键节点的扰动
- 此为 MetaboAnalyst 独有功能

### 2. 六大主流通路库的特性对比

本节概要：本节逐一介绍 KEGG、Reactome、BioCyc/MetaCyc、SMPDB、WikiPathways 和 Human-GEM 六个通路库的覆盖范围、组织方式和收费模式，并给出多维度对比。

通路库是预先整理的结构化生物学通路知识图谱，每条记录通常包含通路名称、化合物列表、反应关系和拓扑结构。

#### 2.1 KEGG Pathway

- 全称：Kyoto Encyclopedia of Genes and Genomes
- 维护方：京都大学（日本）
- 覆盖：约 550 条参考通路（含约 180 条代谢通路），超过 8000 个物种的基因组
- ID 体系：C number（化合物）、R number（反应）、map number（参考通路）、物种特异性通路（如 hsa00010）
- 代谢通路层级：Metabolism 总图 → 碳水化合物代谢（00010-00053，含糖酵解、TCA 循环、磷酸戊糖途径）→ 能量代谢 / 脂质代谢 / 核苷酸代谢 / 氨基酸代谢 / 辅因子与维生素代谢 / 次级代谢物合成
- 优势：使用最广泛、ID 体系完善、与大多数工具兼容
- 劣势：2011 年后 FTP 下载需付费订阅，学术免费版有功能限制

#### 2.2 Reactome

- 全称：Reactome Pathway Database
- 维护方：OICR（加拿大）+ EMBL-EBI（欧洲）
- 覆盖：约 2,600 条人类通路，分层组织
- 优势：完全开源免费、人工 curation 质量高、支持通路层级树浏览
- 劣势：以人为中心，其他物种需通过 orthology 推断

#### 2.3 BioCyc / MetaCyc

- 维护方：SRI International（美国）
- MetaCyc：通用代谢通路参考库（非物种特异），含 3,000+ 条通路
- BioCyc：含 20,000+ 物种/菌株的基因组预测代谢网络
- 优势：代谢通路覆盖最全面，植物和微生物通路尤其丰富
- 劣势：部分高级功能收费

#### 2.4 SMPDB（Small Molecule Pathway Database）

- 维护方：Wishart Lab（加拿大），与 MetaboAnalyst 同一团队
- 覆盖：600+ 条人类小分子信号与代谢通路
- 优势：专为代谢组学设计，包含通路图示和药物通路，与 MetaboAnalyst 深度集成
- 劣势：物种覆盖以人类为主

#### 2.5 WikiPathways

- 维护方：社区驱动（开源 Wiki 模式）
- 覆盖：约 3,000 条通路，覆盖多种物种
- 优势：完全开源、社区快速更新、支持 GPML 格式
- 劣势：质量依赖社区贡献，条目间一致性参差不齐

#### 2.6 Recon3D / Human-GEM

- 全称：Human Metabolic Reconstruction
- 维护方：瑞典 Chalmers 理工大学
- 覆盖：人类全基因组代谢模型，约 13,000 个反应、4,000 个代谢物
- 优势：当前最完整的人类代谢网络，可用于通量平衡分析（Flux Balance Analysis, FBA）等约束建模
- 劣势：无传统通路边界概念，是整体网络模型而非离散通路集合

#### 2.7 六库多维度对比

| 特性 | KEGG | Reactome | BioCyc/MetaCyc | SMPDB | WikiPathways |
|------|------|----------|----------------|-------|--------------|
| 免费程度 | ★★☆ | ★★★ | ★★☆ | ★★★ | ★★★ |
| 代谢通路覆盖 | ★★★ | ★★☆ | ★★★ | ★★★ | ★★☆ |
| 拓扑信息完整度 | ★★★ | ★★★ | ★★★ | ★★☆ | ★★☆ |
| 人工 curation 质量 | ★★★ | ★★★ | ★★★ | ★★★ | ★★☆ |
| 代谢组学专用程度 | ★★☆ | ★★☆ | ★★★ | ★★★ | ★★☆ |
| 物种多样性 | ★★★ | ★☆☆ | ★★★ | ★★☆ | ★★☆ |
| API 友好度 | ★★☆ | ★★★ | ★★☆ | ★★☆ | ★★★ |

选择建议：非靶向代谢组学首选 KEGG（代谢通路层级清晰）或 SMPDB（代谢组学专用）；需要完全开源方案时选择 Reactome；植物/微生物研究优先考虑 BioCyc/MetaCyc；需要代谢网络建模时使用 Human-GEM。

### 3. 背景集在统计检验中的作用与五种选择方案

本节概要：本节说明背景集在 ORA 统计检验中的数学角色，解释为何背景集选择直接影响 P 值，并逐一分析五种常见背景集方案的优缺点与适用条件。

#### 3.1 背景集的统计意义

背景集（Background Set / Reference Set）是通路分析中计算期望概率的参考集合。ORA 的 Fisher 精确检验使用 2×2 列联表：

|  | 属于该通路 | 不属于该通路 | 合计 |
|---|----------|------------|------|
| 差异代谢物 | a | b | D |
| 背景集中非差异代谢物 | c | d | N-D |
| 合计 | P | N-P | N |

N（背景集代谢物总数）的大小直接影响 P 值：N 过大则 P/N 被稀释，显著性被低估（假阴性增多）；N 过小则显著性被高估（假阳性增多）。

#### 3.2 方案一：实验检测到的所有代谢物（Detected Metabolites）

- 定义：实验中所有被检测到并成功注释的代谢物，无论是否差异
- 适用：非靶向代谢组学（untargeted metabolomics）
- 优点：最符合统计假设，"能测到的世界"即为可推断的范围
- 缺点：检测范围受仪器和色谱方法影响，批次间可比性受限
- 评价：MetaboAnalyst 的默认推荐，也是最合理的选择

#### 3.3 方案二：化合物库中的所有代谢物（Library Compounds）

- 定义：代谢物鉴定库中全部有记录的代谢物（如 HMDB 所有化合物）
- 适用：靶向代谢组学，使用特定 panel 检测时
- 优点：背景集稳定、可重复、不受检测条件影响
- 缺点：显著高估"能测到的范围"，导致背景过大、P 值稀释、假阴性增多
- 评价：仅适用于靶向分析且 panel 较小（200-500 代谢物）时

#### 3.4 方案三：通路库中的所有代谢物（Pathway-Associated Compounds）

- 定义：所选通路库（如 KEGG）中包含的所有化合物
- 缺点：KEGG 包含大量外源化合物（xenobiotics）和植物/微生物代谢物，严重偏离人类代谢组实验现实
- 评价：一般不推荐，除非有特定的研究设计理由

#### 3.5 方案四：物种特异性代谢组模型（Species-Specific Metabolome）

- 定义：基于基因组信息预测的该物种全部可能代谢物（如 Human-GEM 中的代谢物集合）
- 优点：理论上的"该物种全部代谢物"上限
- 缺点：包含大量从未被实验检测到的代谢物，背景过大导致 P 值保守
- 评价：适合作为敏感性分析或探索性研究时使用

#### 3.6 方案五：自定义背景集

- 定义：用户手动上传代谢物列表作为背景
- 适用：特定通路群研究（如仅关注脂质通路）、元分析
- 优点：完全可控
- 缺点：需要领域知识支撑选择
- 评价：在有明确生物学假设时使用

#### 3.7 选择原则

核心原则：统计推断只应基于"能观测到的世界"。检测不到某代谢物，它就是不可推断的，不应纳入背景。因此默认首选"实验检测到的所有代谢物"作为背景集。

### 4. MetaboAnalyst 平台架构与通路分析引擎

本节概要：本节说明 MetaboAnalyst 的开发背景、七大功能模块组织以及从数据上传到结果解读的完整工作流，重点解析通路分析模块中的 ORA、QEA 和拓扑分析三层引擎。

#### 4.1 平台概述

- 开发方：Wishart Lab, University of Alberta, Canada
- 当前版本：6.0（截至 2024 年）
- 技术栈：R（后端统计）+ Java（通路分析引擎）+ JavaScript（Web 界面）
- 许可：学术免费，有企业授权方案
- 文献引用量：超过 30,000 次

#### 4.2 七大功能模块

- 统计分析模块：单变量分析（t-test, ANOVA, Wilcoxon, Kruskal-Wallis）、多变量分析（PCA, PLS-DA, sPLS-DA, OPLS-DA）、聚类分析（层次聚类、k-means、热图）、机器学习（Random Forest, SVM）、ROC 曲线与特征选择
- 功能分析模块（核心）：通路富集分析（ORA）、定量富集分析（QEA）、通路拓扑分析、联合通路分析
- 时间序列/双因素分析：ASCA、MEBA、重复测量 ANOVA
- 功效分析：样本量估计、统计功效计算
- 生物标志物分析：ROC 曲线、多变量 ROC、决策树/逻辑回归模型
- 整合分析：代谢组-转录组联合通路分析、知识网络分析
- 数据处理工具：归一化、缺失值填补、数据转换（Log/Pareto scaling）

#### 4.3 通路分析三层引擎

**第一层 — ORA：**

- 用户上传差异代谢物列表（按 VIP > 1、p < 0.05 筛选）
- 系统将代谢物 ID 映射到选定通路库
- 对每条通路执行 Fisher 精确检验
- 多重检验校正后输出富集结果表和气泡图

**第二层 — QEA：**

- 用户上传完整代谢物定量表（含 fold change 或 t-score）
- 使用 Globaltest 或 GlobalAncova 算法
- 将代谢物按变化幅度排序后检验通路整体扰动
- 比 ORA 更灵敏，适用于发现效应较小但协同变化的通路

**第三层 — 通路拓扑分析：**

- 在富集分析基础上叠加通路网络拓扑信息
- 计算每个命中代谢物在通路中的中心性得分
- 综合 Pathway Impact 和 p-value 给出最终评分
- 通过气泡图直观展示通路的重要性和显著性

#### 4.4 完整分析工作流

数据上传（浓度矩阵 + 实验设计表 + 选择物种/通路库）→ 数据预处理（缺失值填补 → 低质量特征过滤 → 归一化）→ 数据探索与差异分析（PCA 观察组间分离 → PLS-DA 筛选关键代谢物 → 单变量检验 → VIP + p-value 筛选差异代谢物）→ 通路分析（ORA 初筛 → QEA 定量验证 → 拓扑分析确定关键通路）→ 结果解读（显著通路列表 → 通路网络可视化 → KEGG 通路图高亮 → 生物学假设生成）

### 5. MetaboAnalyst 归一化方法、限制与其他工具

本节概要：本节列出 MetaboAnalyst 提供的七种归一化方法及其适用场景，说明平台的六项使用限制，并简要对比其他代谢组学通路分析工具。

#### 5.1 归一化方法

| 方法 | 缩写 | 适用场景 | 原理 |
|------|------|----------|------|
| 中位数归一化 | Median | 广泛适用 | 每个样本除以该样本的代谢物中位数 |
| 分位数归一化 | Quantile | 消除样本间分布差异 | 强制所有样本具有相同分布 |
| 概率商归一化 | PQN | NMR 数据首选 | 计算每个代谢物的样本/参考比值 |
| 总离子流归一化 | TIC | LC-MS/GC-MS | 每个代谢物除以该样本总强度 |
| 方差稳定化归一化 | VSN | 微阵列/LC-MS | 方差稳定化变换 |
| MSTUS | MSTUS | 推荐默认 | 使用无变化的代谢物子集进行归一化 |
| Pareto Scaling | — | PLS-DA 前 | 均值中心化后除以标准差平方根 |

#### 5.2 使用限制

- 免费版有样本数和代谢物数上传上限
- 通路分析默认以 Homo sapiens 为基础，其他物种支持有限
- 代谢物名称到 KEGG/HMDB ID 的映射依赖内置数据库，部分代谢物可能映射失败
- 多重检验校正方法的选择（FDR vs Bonferroni vs 无校正）显著影响结果
- 通路库中的通路边界是人为划分的，实际生物网络中通路是连续的
- 代谢物鉴定错误会导致通路分析结果无意义（garbage in, garbage out）

#### 5.3 其他代谢组学通路分析工具

| 工具 | 核心特点 | 与 MetaboAnalyst 的差异 |
|------|----------|------------------------|
| MBRole 2.0 | 基于化学相似性（ChemRICH）而非传统通路定义 | 不需要 ID 映射，适合非靶向分析 |
| Mummichog | 直接从 LC-MS peaks（m/z）预测通路，不依赖代谢物鉴定 | 适合探索性分析，MetaboAnalyst 已集成 |
| MetExplore | 代谢网络可视化与通路映射 | 可视化更强，统计分析较弱 |
| iPath 3.0 | KEGG 全局代谢通路可视化 | 仅可视化，不做统计分析 |
| Paintomics | 多组学通路可视化 | 多组学整合是强项 |
| MetPA | MetaboAnalyst 旧版 Python 独立包 | 功能已被 MetaboAnalyst 取代 |
