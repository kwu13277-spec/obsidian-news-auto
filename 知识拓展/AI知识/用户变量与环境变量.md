### 1. 变量的基本含义：给系统和程序看的“信息标签”

本节概要：本节说明变量在操作系统中的基本作用，重点是理解它不是普通文件，而是系统或程序读取的一类配置信息。

在计算机中，变量可以理解为一组“名称 = 值”的信息。例如：

```text
PATH=C:\Program Files\Python312\Scripts\;C:\Program Files\Python312\
TEMP=C:\Users\Kingwu\AppData\Local\Temp
USERNAME=Kingwu
```

这些变量不是给人直接使用的，而是给操作系统、命令行工具和软件程序读取的。程序运行时，经常需要知道一些基础信息，例如用户是谁、临时文件放在哪里、Python 安装在哪里、某个软件的可执行文件路径在哪里。环境变量（Environment Variables）就是用来保存这些信息的。

可以把环境变量理解成操作系统提供给程序的一张“配置表”。程序启动时，会读取这张表，决定自己该去哪里找文件、调用哪个工具、使用什么路径、按照什么规则运行。

<a id="section-2"></a>

### 2. 用户变量与系统环境变量的区别：作用范围不同

本节概要：本节区分用户变量和系统变量，重点是说明二者的生效范围、权限要求和适合使用的场景。

在 Windows 中，环境变量通常分为两类：用户变量和系统变量。它们本质上都是环境变量，但作用范围不同。

| 类型 | 英文名称 | 作用范围 | 是否影响其他用户 | 是否通常需要管理员权限 | 适合场景 |
|---|---|---|---|---|---|
| 用户变量 | User variables | 只对当前 Windows 用户生效 | 不影响 | 通常不需要 | 个人安装的软件、个人开发环境 |
| 系统变量 | System variables | 对整台电脑所有用户生效 | 影响 | 通常需要 | 全局软件、系统级工具、多人共用环境 |

用户变量只属于当前登录的 Windows 账户。例如你用 `Kingwu` 这个账户登录电脑，那么用户变量只对 `Kingwu` 生效。如果电脑上还有另一个用户账户，他通常不会受到你用户变量设置的影响。

系统变量属于整个 Windows 系统。只要某个变量写入系统变量，所有用户、系统服务和大多数程序都可能读取到它。因此，系统变量影响范围更大，也更需要谨慎修改。

对于初学者来说，个人电脑上配置 Python、Anaconda、VSCode、Git、Java、R、Node.js 等工具时，如果只是自己使用，优先修改“用户变量”通常更安全；只有当软件明确要求全局可用，或多个用户都要使用时，才考虑修改“系统变量”。

<a id="section-3"></a>

### 3. PATH 变量如何影响命令执行：系统通过 PATH 找到程序位置

本节概要：本节说明 PATH 是最常见也最容易出错的环境变量，重点是理解命令行为什么能直接识别 python、git、conda 等命令。

PATH 是最重要、最常见的环境变量之一。它保存了一组文件夹路径。命令行输入一个命令时，系统会按照 PATH 中记录的路径顺序，逐个文件夹寻找对应的可执行程序。

例如你在终端输入：

```bash
python
```

Windows 并不是天然知道 `python` 在哪里。它会去 PATH 中记录的文件夹里查找：

```text
C:\Users\Kingwu\anaconda3\
C:\Users\Kingwu\anaconda3\Scripts\
C:\Program Files\Git\cmd\
C:\Windows\System32\
```

如果某个文件夹里存在 `python.exe`，系统就会调用它。如果 PATH 里没有 Python 所在目录，终端就可能提示：

```text
'python' 不是内部或外部命令，也不是可运行的程序或批处理文件。
```

PATH 的核心作用可以概括为一句话：让你在任何目录下输入命令时，系统都能找到对应程序。

这就是为什么安装 Python、Git、Anaconda、Java、R 等工具时，经常会看到“Add to PATH”或“添加到环境变量”的选项。勾选这个选项后，软件安装程序会把自己的可执行文件目录写入 PATH。

<a id="section-4"></a>

### 4. 常见变量类型与使用场景：不只有 PATH

本节概要：本节整理常见环境变量的类型，重点是说明它们分别用于路径定位、用户信息、临时目录和软件配置。

环境变量不只有 PATH。不同变量保存的信息不同，常见类型包括路径变量、用户信息变量、临时目录变量和软件专用变量。

| 变量名 | 常见含义 | 典型用途 |
|---|---|---|
| PATH | 可执行程序搜索路径 | 让终端找到 python、git、conda 等命令 |
| TEMP / TMP | 临时文件夹路径 | 程序运行时存放临时文件 |
| USERNAME | 当前用户名 | 程序识别当前 Windows 用户 |
| USERPROFILE | 当前用户主目录 | 定位 `C:\Users\用户名` |
| JAVA_HOME | Java 安装目录 | Java、Maven、Gradle 等工具查找 JDK |
| PYTHONPATH | Python 模块搜索路径 | Python 查找额外模块位置 |
| CONDA_PREFIX | 当前 Conda 环境路径 | 识别当前激活的 Conda 环境 |

其中，PATH 是给系统找“命令程序”用的；JAVA_HOME、PYTHONPATH、CONDA_PREFIX 等变量更多是给特定软件生态使用的。不同变量有不同规则，不能把所有路径都随便放进 PATH。

<a id="section-5"></a>

### 5. 配置变量的基本方法：图形界面与命令行两种方式

本节概要：本节说明 Windows 中配置环境变量的常见入口，重点是区分图形界面修改和命令行临时修改。

在 Windows 中，最常用的方法是通过系统图形界面修改环境变量。路径通常是：

```text
此电脑 → 属性 → 高级系统设置 → 环境变量
```

进入后会看到两个区域：上方是“用户变量”，下方是“系统变量”。如果你只是配置自己使用的软件，通常优先在“用户变量”里修改 PATH 或添加新变量。

修改 PATH 时，不是把所有内容写成一大串，而是点击“编辑”，然后逐条添加文件夹路径。例如：

```text
C:\Users\Kingwu\anaconda3
C:\Users\Kingwu\anaconda3\Scripts
C:\Users\Kingwu\anaconda3\Library\bin
```

也可以在命令行中查看变量。例如：

```cmd
where python
where git
echo %PATH%
```

在 PowerShell 中常用：

```powershell
Get-Command python
$env:PATH
```

需要注意的是，命令行中临时设置的环境变量通常只在当前终端窗口中生效。关闭窗口后，临时设置可能消失。图形界面中修改的变量一般是持久生效的，但通常需要重新打开终端或重启软件才能被新进程读取。

### 6. 初学者常见误区：变量配置错误会导致命令找不到或调用错版本

本节概要：本节说明环境变量配置中最常见的问题，重点是 PATH 顺序、路径是否写对、终端是否重启和多版本冲突。

初学者最常见的问题不是没有安装软件，而是安装了软件但系统找不到，或者系统找到了错误版本的软件。

常见误区包括：

- 把文件路径写进 PATH，而不是文件夹路径。PATH 里通常应放 `python.exe` 所在的文件夹，而不是 `python.exe` 文件本身。
- 修改环境变量后没有重新打开终端。旧终端通常读取的是修改前的变量。
- 用户变量和系统变量里都有同类路径，导致调用顺序混乱。
- 同时安装了 Python、Anaconda、Microsoft Store Python，结果 `python` 命令指向了不想用的版本。
- PATH 中旧版本路径排在新版本路径前面，导致终端优先调用旧版本。
- 删除系统 PATH 中的重要路径，例如 `C:\Windows\System32`，可能导致系统命令异常。

检查命令实际调用位置时，不要只看软件是否安装成功，而要看终端到底调用了哪个路径。例如：

```cmd
where python
where conda
where git
```

如果输出多个路径，通常第一个路径会被优先调用。PATH 的顺序因此非常重要。

<a id="section-7"></a>

### 7. 简明判断原则：个人软件优先用户变量，全局工具才改系统变量

本节概要：本节给出实际判断方法，重点是帮助初学者在配置软件环境时减少误改系统变量的风险。

判断应该修改用户变量还是系统变量，可以按照下面的原则处理。

| 场景 | 建议修改位置 | 原因 |
|---|---|---|
| 只有自己使用 Python、Anaconda、Git、VSCode | 用户变量 | 影响范围小，风险低 |
| 多个 Windows 用户都要使用同一个工具 | 系统变量 | 所有用户都能访问 |
| 软件安装教程明确要求配置系统变量 | 视情况配置系统变量 | 某些开发工具需要全局识别 |
| 不确定是否需要全局生效 | 优先用户变量 | 可控、易恢复 |
| 修改系统关键路径 | 谨慎操作 | 误删可能影响系统命令 |

最实用的理解方式是：

```text
用户变量 = 只给当前用户看的配置
系统变量 = 给整台电脑和所有用户看的配置
PATH = 告诉系统去哪些文件夹找命令
```

如果你只是为了让自己的终端能运行 `python`、`conda`、`git`、`code` 等命令，通常只需要把对应软件目录加入当前用户的 PATH。配置完成后，重新打开终端，再用 `where python`、`where git` 这类命令确认系统实际调用的位置。
